# Container Tokens

TENsor provides 5 semantic container size tokens for consistent, responsive layout design. Introduced in Phase 1 as part of foundation consolidation.

## Container Sizes

Use container tokens to define maximum widths for responsive layouts:

```typescript
import { tokens } from '@tenbaselabs/tensor';

tokens.container.sm   // 640px - Small devices, mobile landscape
tokens.container.md   // 768px - Tablets
tokens.container.lg   // 1024px - Desktop
tokens.container.xl   // 1280px - Large desktop
tokens.container.full // 100% - Full width, no constraint
```

## Usage Guidelines

### Semantic Breakpoints

| Token | Width | Device Target | Use Case |
|-------|-------|---------------|----------|
| `sm` | 640px | Mobile landscape, small tablets | Narrow content, forms |
| `md` | 768px | Tablets (portrait) | Standard content width |
| `lg` | 1024px | Desktop, tablets (landscape) | Dashboard layouts, wide content |
| `xl` | 1280px | Large desktop displays | Maximum content width |
| `full` | 100% | All devices | Full-width sections, sidebars |

## Common Patterns

### Page Container

```tsx
// Standard page layout with max-width constraint
<div style={{
  maxWidth: tokens.container.lg,
  marginLeft: 'auto',
  marginRight: 'auto',
  padding: tokens.spacing.md,
}}>
  <h1>Page Content</h1>
  <p>Constrained to 1024px on large screens</p>
</div>
```

### Responsive Container

```tsx
// Mobile-first responsive container
<div style={{
  maxWidth: tokens.container.sm,    // Mobile: 640px
  '@media (min-width: 768px)': {
    maxWidth: tokens.container.md,  // Tablet: 768px
  },
  '@media (min-width: 1024px)': {
    maxWidth: tokens.container.lg,  // Desktop: 1024px
  },
}}>
  Content adapts to screen size
</div>
```

Or with Tailwind CSS:

```tsx
<div className="max-w-sm md:max-w-md lg:max-w-lg mx-auto">
  Responsive container with Tailwind
</div>
```

### Full-Width Section

```tsx
// Hero section that spans full width
<section style={{
  maxWidth: tokens.container.full,
  backgroundColor: tokens.colors.surface.raised,
  padding: `${tokens.spacing['2xl']} ${tokens.spacing.md}`,
}}>
  <div style={{
    maxWidth: tokens.container.lg,
    marginLeft: 'auto',
    marginRight: 'auto',
  }}>
    <h1>Hero content constrained within full-width background</h1>
  </div>
</section>
```

### Nested Containers

```tsx
// Outer container for full-width background
<div style={{ maxWidth: tokens.container.full }}>
  {/* Inner container for content constraint */}
  <div style={{
    maxWidth: tokens.container.md,
    marginLeft: 'auto',
    marginRight: 'auto',
    padding: tokens.spacing.lg,
  }}>
    <article>Centered, constrained content</article>
  </div>
</div>
```

## Media Query Integration

Container tokens work seamlessly with media queries:

```tsx
const containerStyle = {
  width: '100%',
  maxWidth: tokens.container.sm,
  marginLeft: 'auto',
  marginRight: 'auto',
  padding: tokens.spacing.md,
};

// In CSS-in-JS
const styles = `
  .container {
    width: 100%;
    max-width: ${tokens.container.sm};
    margin-left: auto;
    margin-right: auto;
  }

  @media (min-width: ${tokens.container.md}) {
    .container {
      max-width: ${tokens.container.md};
    }
  }

  @media (min-width: ${tokens.container.lg}) {
    .container {
      max-width: ${tokens.container.lg};
    }
  }
`;
```

## Responsive Layout Examples

### Dashboard Grid

```tsx
// Responsive grid that adapts to container width
<div style={{
  maxWidth: tokens.container.xl,
  marginLeft: 'auto',
  marginRight: 'auto',
  display: 'grid',
  gridTemplateColumns: 'repeat(auto-fit, minmax(300px, 1fr))',
  gap: tokens.spacing.lg,
  padding: tokens.spacing.md,
}}>
  <Card />
  <Card />
  <Card />
</div>
```

### Article Layout

```tsx
// Narrow container for optimal reading experience
<article style={{
  maxWidth: tokens.container.md,
  marginLeft: 'auto',
  marginRight: 'auto',
  padding: `${tokens.spacing['2xl']} ${tokens.spacing.md}`,
}}>
  <h1 style={{
    fontSize: tokens.typography.fontSize['3xl'],
    marginBottom: tokens.spacing.lg,
  }}>
    Article Title
  </h1>
  <p style={{
    fontSize: tokens.typography.fontSize.base,
    lineHeight: tokens.typography.lineHeight.relaxed,
  }}>
    Long-form content with comfortable line length
  </p>
</article>
```

### Sidebar Layout

```tsx
// Two-column layout with sidebar
<div style={{
  maxWidth: tokens.container.xl,
  marginLeft: 'auto',
  marginRight: 'auto',
  display: 'grid',
  gridTemplateColumns: '250px 1fr',
  gap: tokens.spacing.lg,
}}>
  <aside style={{ maxWidth: tokens.container.full }}>
    Sidebar (250px fixed)
  </aside>
  <main style={{ maxWidth: tokens.container.full }}>
    Main content (flexible)
  </main>
</div>
```

## Do's and Don'ts

### Do's

✅ Use semantic container tokens for max-width
```tsx
maxWidth: tokens.container.lg  // ✅
```

✅ Combine with auto margins for centering
```tsx
style={{
  maxWidth: tokens.container.md,
  marginLeft: 'auto',
  marginRight: 'auto',
}}
```

✅ Use full for sections that need full width
```tsx
<section style={{ maxWidth: tokens.container.full }}>
  Full-width background
</section>
```

✅ Match container breakpoints with media queries
```tsx
@media (min-width: ${tokens.container.md}) {
  // Tablet styles
}
```

### Don'ts

❌ Don't hard-code container widths
```tsx
// Bad
maxWidth: '1024px'

// Good
maxWidth: tokens.container.lg
```

❌ Don't use arbitrary breakpoints
```tsx
// Bad
@media (min-width: 900px) { }

// Good
@media (min-width: ${tokens.container.md}) { }
```

❌ Don't constrain containers too narrowly on mobile
```tsx
// Bad - too narrow on small screens
<div style={{
  maxWidth: tokens.container.sm,
  padding: tokens.spacing.md,
}}>
  // Only 640px - 32px = 608px usable space
</div>

// Good - full width on mobile, constrained on larger screens
<div style={{
  maxWidth: tokens.container.full,
  '@media (min-width: 640px)': {
    maxWidth: tokens.container.sm,
  },
}}>
</div>
```

## Accessibility Considerations

### Reading Width

For long-form content, constrain line length to 60-80 characters for optimal readability:

```tsx
<article style={{
  maxWidth: tokens.container.md,  // 768px
  marginLeft: 'auto',
  marginRight: 'auto',
}}>
  <p style={{ maxWidth: '65ch' }}>
    Content with comfortable reading width
  </p>
</article>
```

### Touch Targets

Ensure touch targets remain accessible even in constrained containers:

```tsx
<button style={{
  padding: `${tokens.spacing.sm} ${tokens.spacing.md}`,
  // Minimum 44px height for touch accessibility
  minHeight: '44px',
}}>
  Accessible Button
</button>
```

## Migration from Hard-Coded Widths

Replace arbitrary max-width values with semantic container tokens:

```diff
// Before
- <div style={{ maxWidth: '1024px' }}>
+ <div style={{ maxWidth: tokens.container.lg }}>

// Before
- @media (min-width: 768px) { }
+ @media (min-width: ${tokens.container.md}) { }
```
