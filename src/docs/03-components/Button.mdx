# Button

Buttons trigger actions and events in response to user interaction. Use buttons for actions like submitting forms, opening modals, or starting processes.

## When to Use

✅ **Use Button when:**
- Submitting a form or saving data
- Opening a modal or dialog
- Triggering an immediate action (delete, confirm, start process)
- Toggling application state

❌ **Don't use Button when:**
- Navigating to a different page — use `Link` instead
- Showing/hiding content without an action — use disclosure pattern
- The element is not interactive — use appropriate semantic HTML

## Installation

```tsx
import { Button } from '@tenbaselabs/tensor-react';
```

## API Reference

### Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `variant` | `'primary' \| 'secondary' \| 'danger' \| 'ghost'` | `'primary'` | Visual style variant. Use `primary` for main actions, `secondary` for alternatives, `danger` for destructive actions, `ghost` for minimal emphasis. |
| `size` | `'sm' \| 'md' \| 'lg'` | `'md'` | Button size. `sm` (36px), `md` (44px), `lg` (56px). All meet WCAG touch target requirements. |
| `loading` | `boolean` | `false` | Shows spinner and disables interaction. Use for async operations to prevent double-submission. |
| `disabled` | `boolean` | `false` | Disables button interaction. Sets `disabled` attribute and `aria-disabled="true"`. |
| `type` | `'button' \| 'submit' \| 'reset'` | `'button'` | HTML button type. Use `submit` for form submission, `button` for other actions. |
| `onClick` | `(event: MouseEvent) => void` | `undefined` | Click event handler. Called when button is clicked or activated via keyboard. |
| `aria-label` | `string` | `undefined` | Accessible label. Required for icon-only buttons without visible text. |
| `children` | `ReactNode` | **required** | Button content (text, icon, or both). |
| `className` | `string` | `undefined` | Additional CSS classes for custom styling. |

### TypeScript Interface

```typescript
import { type ButtonProps } from '@tenbaselabs/tensor-react';

interface ButtonProps extends ButtonHTMLAttributes<HTMLButtonElement> {
  variant?: 'primary' | 'secondary' | 'danger' | 'ghost';
  size?: 'sm' | 'md' | 'lg';
  loading?: boolean;
  'aria-label'?: string;
  children: React.ReactNode;
}
```

## Usage Examples

### Basic Button

```tsx
import { Button } from '@tenbaselabs/tensor-react';

function Example() {
  return (
    <Button onClick={() => console.log('clicked')}>
      Click me
    </Button>
  );
}
```

### Variants

#### Primary (Default)

Use for the main call-to-action. Limit to one primary button per section.

```tsx
<Button variant="primary">
  Save Changes
</Button>
```

#### Secondary

Use for alternative actions with less emphasis.

```tsx
<Button variant="secondary">
  Cancel
</Button>
```

#### Danger

Use for destructive actions. Always pair with confirmation for irreversible changes.

```tsx
<Button variant="danger">
  Delete Account
</Button>
```

#### Ghost

Use for tertiary actions with minimal visual weight.

```tsx
<Button variant="ghost">
  Learn More
</Button>
```

### Sizes

```tsx
// Small (36px height) — Compact UIs, tables, inline actions
<Button size="sm">Small Button</Button>

// Medium (44px height, default) — Standard forms, cards
<Button size="md">Medium Button</Button>

// Large (56px height) — Hero CTAs, primary landing actions
<Button size="lg">Large Button</Button>
```

### Loading State

Show loading feedback during async operations:

```tsx
import { useState } from 'react';
import { Button } from '@tenbaselabs/tensor-react';

function SaveButton() {
  const [isSaving, setIsSaving] = useState(false);

  const handleSave = async () => {
    setIsSaving(true);
    try {
      await saveData();
    } finally {
      setIsSaving(false);
    }
  };

  return (
    <Button loading={isSaving} onClick={handleSave}>
      {isSaving ? 'Saving...' : 'Save Changes'}
    </Button>
  );
}
```

### Disabled State

```tsx
<Button disabled>
  Unavailable Action
</Button>

// Conditional disable
<Button disabled={!formValid}>
  Submit Form
</Button>
```

### Form Buttons

```tsx
<form onSubmit={handleSubmit}>
  <input type="text" name="email" />

  {/* Submit button */}
  <Button type="submit" variant="primary">
    Submit
  </Button>

  {/* Reset button */}
  <Button type="reset" variant="secondary">
    Reset Form
  </Button>

  {/* Regular button (won't submit) */}
  <Button type="button" variant="ghost" onClick={handleCancel}>
    Cancel
  </Button>
</form>
```

### Icon-Only Button

When using an icon without text, provide an accessible label:

```tsx
import { XIcon } from '@tenbaselabs/icons';

<Button
  variant="ghost"
  size="sm"
  aria-label="Close dialog"
>
  <XIcon className="h-4 w-4" />
</Button>
```

### Button with Icon

```tsx
import { PlusIcon } from '@tenbaselabs/icons';

<Button variant="primary">
  <PlusIcon className="mr-2 h-5 w-5" />
  Add Item
</Button>
```

## Choosing a Variant

| Variant | When to Use | Example Actions |
|---|---|---|
| **Primary** | Main call-to-action, highest priority action on the page | "Save", "Submit", "Continue", "Sign Up" |
| **Secondary** | Alternative actions, supporting actions with less emphasis | "Cancel", "Back", "Skip", "Close" |
| **Danger** | Destructive or irreversible actions that require caution | "Delete", "Remove", "Deactivate", "Reset" |
| **Ghost** | Tertiary actions, minimal visual weight, supplementary actions | "Learn More", "Dismiss", "View Details" |

**Best Practice:** Limit primary buttons to one per section to maintain clear action hierarchy.

## Accessibility

### ARIA Attributes

The Button component automatically manages ARIA attributes:

```tsx
// Loading state
<Button loading>Save</Button>
// Renders with: aria-busy="true" disabled

// Disabled state
<Button disabled>Submit</Button>
// Renders with: disabled aria-disabled="true"

// Icon-only button (requires aria-label)
<Button aria-label="Close">
  <CloseIcon />
</Button>
```

### Keyboard Navigation

| Key | Action |
|---|---|
| `Tab` | Moves focus to button (or next focusable element) |
| `Shift + Tab` | Moves focus to previous focusable element |
| `Enter` | Activates button, triggers `onClick` |
| `Space` | Activates button, triggers `onClick` |

### Focus Management

All buttons have a visible focus indicator meeting WCAG 2.1 guidelines:
- 2px outline with 3:1 contrast ratio against background
- 2px offset for clarity
- Outline color matches variant semantic color

After async actions complete, focus remains on the button (not removed programmatically).

### Screen Reader Announcements

| State | Announcement |
|---|---|
| Default | "Save Changes, button" |
| Disabled | "Save Changes, button, dimmed" or "unavailable" |
| Loading | "Save Changes, button, busy" |
| Icon-only | "[aria-label value], button" |

### Color Contrast (WCAG AAA)

All button variants meet WCAG 2.1 Level AAA contrast requirements:

| Variant | Contrast Ratio | WCAG Level |
|---|---|---|
| Primary | 8.2:1 (white on blue) | AAA ✅ |
| Secondary | 15.1:1 (dark gray on light gray) | AAA ✅ |
| Danger | 5.9:1 (white on red) | AAA ✅ |
| Ghost | 16.1:1 (dark gray on white) | AAA ✅ |

## Best Practices

### Do's

✅ **Use descriptive, action-oriented text**
```tsx
<Button>Save Changes</Button>  // Clear
<Button>Delete Account</Button>  // Specific
```

✅ **Provide loading feedback for async actions**
```tsx
<Button loading={isSubmitting}>
  {isSubmitting ? 'Saving...' : 'Save Changes'}
</Button>
```

✅ **Use semantic button types**
```tsx
<form>
  <Button type="submit">Submit Form</Button>
  <Button type="button">Cancel</Button>  {/* Won't submit */}
</form>
```

✅ **Maintain action hierarchy**
```tsx
<div className="flex gap-2 justify-end">
  <Button variant="ghost">Skip</Button>
  <Button variant="secondary">Back</Button>
  <Button variant="primary">Continue</Button>  {/* Primary on right */}
</div>
```

✅ **Use confirmation for destructive actions**
```tsx
<Button
  variant="danger"
  onClick={() => {
    if (confirm('This will permanently delete your account. Continue?')) {
      deleteAccount();
    }
  }}
>
  Delete Account
</Button>
```

### Don'ts

❌ **Don't use vague text**
```tsx
// Bad
<Button>Click Here</Button>
<Button>Submit</Button>  // Submit what?

// Good
<Button>Delete Account</Button>
<Button>Submit Registration</Button>
```

❌ **Don't use buttons for navigation**
```tsx
// Bad — buttons are for actions, not navigation
<Button onClick={() => window.location.href = '/about'}>
  About Us
</Button>

// Good — use Link for navigation
<Link href="/about">About Us</Link>
```

❌ **Don't nest interactive elements**
```tsx
// Bad — invalid HTML
<Button>
  <a href="/link">Nested link</a>
</Button>

// Good
<Link href="/link">Navigate</Link>
```

❌ **Don't use too many primary buttons**
```tsx
// Bad — unclear priority
<div>
  <Button variant="primary">Action 1</Button>
  <Button variant="primary">Action 2</Button>
  <Button variant="primary">Action 3</Button>
</div>

// Good — clear hierarchy
<div>
  <Button variant="ghost">Secondary Action</Button>
  <Button variant="secondary">Alternative</Button>
  <Button variant="primary">Primary Action</Button>
</div>
```

❌ **Don't rely on color alone**
```tsx
// Bad — no visual distinction beyond color
<div>
  <Button variant="primary">Action A</Button>
  <Button variant="danger">Action B</Button>
</div>

// Good — clear labeling
<div>
  <Button variant="primary">Save Changes</Button>
  <Button variant="danger">Delete Account</Button>
</div>
```

## Common Patterns

### Button Group (Modal Actions)

```tsx
<div className="flex gap-3 justify-end">
  <Button variant="secondary" onClick={onCancel}>
    Cancel
  </Button>
  <Button variant="danger" onClick={onDelete} loading={isDeleting}>
    {isDeleting ? 'Deleting...' : 'Delete'}
  </Button>
</div>
```

### Form Submit with Validation

```tsx
function FormExample() {
  const [formValid, setFormValid] = useState(false);
  const [submitting, setSubmitting] = useState(false);

  return (
    <form onSubmit={handleSubmit}>
      <input onChange={validateForm} />

      <Button
        type="submit"
        disabled={!formValid}
        loading={submitting}
      >
        {submitting ? 'Submitting...' : 'Submit'}
      </Button>
    </form>
  );
}
```

### Confirmation Dialog

```tsx
import { Modal, Button } from '@tenbaselabs/tensor-react';

function DeleteConfirmation({ isOpen, onClose, onConfirm }) {
  const [isDeleting, setIsDeleting] = useState(false);

  const handleDelete = async () => {
    setIsDeleting(true);
    await onConfirm();
    setIsDeleting(false);
    onClose();
  };

  return (
    <Modal isOpen={isOpen} onClose={onClose}>
      <h2>Confirm Deletion</h2>
      <p>This action cannot be undone.</p>

      <div className="flex gap-2 justify-end mt-4">
        <Button variant="secondary" onClick={onClose} disabled={isDeleting}>
          Cancel
        </Button>
        <Button variant="danger" onClick={handleDelete} loading={isDeleting}>
          {isDeleting ? 'Deleting...' : 'Delete'}
        </Button>
      </div>
    </Modal>
  );
}
```

## Theming

Button colors are controlled by design tokens and automatically adapt to theme changes:

```tsx
// TENbase theme
<Button variant="primary">Button</Button>  // Blue background

// PlusOne theme
<Button variant="primary">Button</Button>  // Orange background (brand color)
```

No code changes required — components automatically adapt when theme changes.
